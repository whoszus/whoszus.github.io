​ 目前从事的研究工作中涉及图形图像的内容比较少。经过一段时间的思考和调查，如果要在数字图像领域做一些工作的话，基于图像的骨骼畸形、肌肉畸形的辅助诊断会是我考虑选择的方向。

​ 图1：脊柱侧弯示意图

​ 骨骼畸形，例如脊柱侧弯、骨盆不正等在前期是比较难以发现的，如图1所示，左边是正常的脊柱而右边是一个发生了侧弯的脊柱示意图。在青少年时期由于不良的运动方式和日常行为（坐姿、睡姿和行走等）可能引起早起的症状。然而，轻微的症状很难引起重视，出现症状之后又非常难以康复。并且，还会导致非常多的危害，如下所示：

1. **影响脊柱外形和功能： **脊柱侧弯导致脊柱变形、肩背部不平、胸廓畸形、骨盆倾斜、长短腿、姿势不良等异常形态，同时影响活动度等功能。
 2. **影响生理健康**：脊柱变形易导致肩背部、腰部顽固性疼痛，严重者甚至出现神经受损、神经受压、肢体感觉障碍、下肢麻木、大小便异常等症状。
 3. **影响心肺功能**：早发性脊柱侧弯患者的尸检结果表明，其肺泡数量低于正常人，肺泡过度充气或萎缩，累积肺叶或全肺，肺动脉直径也远低于同龄人。脊柱侧弯患者胸腔容积减小，吸气相和呼气相胸廓容积均低于正常对照组。脊柱侧弯影响气体交换，包括局部通气、血流、通气血流比、弥散等。易发短气、喘气等呼吸障碍，且影响血液循环。
 4. **影响胃肠系统**：脊柱侧弯使腹腔容积减小、脊柱神经对内脏的调节功能紊乱，进而引起食欲不振、消化不良等胃肠系统反应。

目前的诊断方式：

  1. 根据脊突线判断：患者自然站立并弯腰90度，检查者用拇指和示指在患者脊突上从上至下快速准确地压划，受压出发红，可见一条连续的脊突线。可根据该线大致判断是否侧凸、侧凸部位、侧凸方向。

    ​ 图2： 根据脊突线判断

  2. 根据形态改变判断：脊柱侧弯早期表现有脊柱偏离中线，高低肩，肩胛骨高低不一，一侧胸部出现皱褶皮纹等。

    ​ 图3：根据形态改变判断

  3. 根据影像学证据判断：一般借助X片就可以区别侧凸的原因、分类、弯度、部位、旋转、骨龄、代偿度等。CT扫描和MRI则可做更近一步的显示。

    ​ 图4： X光下的脊柱侧弯

基于图形图像的研究现状

​ 已经有研究提出了基于激光三维点云，通过固定激光点的位置获取形态数据的方法做了脊柱侧弯检测的方法，但由于设备价格高、测试时需要到专门的场地、对不同身高的测试者需要调整激光位置等不理因素，并没有取得很好的成果。同时，由于数字图像的研究已经较为可靠，已经有研究从上述的三个诊断手段提出了基于图像的研究。

  1. 莫尔条纹筛查系统用于对患者的脊椎进行医学检查。该设备用于捕获波纹图像，以基于波纹图像中的波纹图案的对称度来筛选脊柱畸形。之所以使用云纹系统，是因为如果不加以治疗，青春期特发性脊柱侧弯（AIS）可能会导致严重的健康问题。因此，由于监视和早期检测对于防止此问题至关重要。此外，机器学习中的图像识别，尤其是卷积神经网络（CNN），在过去几年中取得了令人瞩目的成就。 [3-4]

​ 图5：利用CNN，基于莫尔条纹进行检查

  1. 利用脊柱前后X射线CT成像是一种非常有效果的诊断工具，可帮助诊断和阐明青少年特发性脊柱侧凸（AIS）评估。[14]提出了一种从X射线CT图像进行AIS评估的自动检测方法。同时，[5] 提出了一种新颖的多视图关联网络（MVC-Net）体系结构，该体系结构可以为多视图（AP和LAT）X射线中的脊柱弯曲估计提供全自动的端到端框架。拟议的MVC-Net使用我们新设计的多视图卷积层来合并多视图X射线的联合特征，从而允许网络通过利用两个视图的结构依赖性来减轻遮挡问题。

  1. 深度学习算法的应用具有减少脊柱侧弯筛查中不必要的转诊和成本的潜力。[7] 开发并验证了深度学习算法，该算法可使用裸露的背部图像进行自动脊柱侧弯检查。该算法的准确性在检测脊柱侧弯，检测曲线≥20°的病例以及对二元分类和四分类的严重性分级方面优于人类专家。计算机视觉由于其在图像信息提取中的良好性能而被公认为是一种有前景的医学图像识别方法[8-10]。通过计算机视觉已经完成了许多任务，包括对眼部疾病的自动分类[11]，识别遗传性疾病的面部表型[12]以及根据眼底图像预测心血管危险因素[13]。当人类筛选者评估外观特征以检测脊柱侧弯时，计算机视觉也可以识别这些特征。在这里，[7]使用裸露的背面图像，开发了深度学习算法，并验证了它们在脊柱侧弯的检测和严重度分级中的可行性和有效性。

​ 图6： 利用深度学习，使用裸露背部的照片进行诊断

粗略的课程学习目标

​ 由于对机器视觉只有浅薄的认知，所以更期待去了解算法或者图像处理中的原理，特别是基础理论、图像处理中基本的原则和trade-off。我非常同意老师提到的只有了解最基本的原理才能在有好的成果，因此，了解原理是我首要的目标。其次就是了解图像领域的最新研究动态，当然通过自己做检索也能查到，但是我相信老师多年的经验积累，对研究进展的理解和视野的高度更高、深度更深。

##MPI

在程序中，不同的进程需要相互的数据交换，特别是在科学计算中，需要大规模的计算与数据交换，集群可以很好解决单节点计算力不足的问题，但在集群中大规模的数据交换是很耗费时间的，因此需要一种在多节点的情况下能快速进行数据交流的标准，这就是MPI。

mpi 只是一个网络传输接口，和 tensorflow, mxnet, spark 都不是一个层面的技术。换句话说，mpi 的对比对象应该是 zmq 或者底层 socket 这样的网络接口。我们可以使用类似于 mpi 或者 zmq 这样的接口去实现这些系统。

MPI定义的是接口，具体用的时候我们是用某个特定的实现，例如openmpi或者mpich2. 为了简单，这里统一叫MPI。

现状：MPI在高性能计算界(特别是成熟的科学计算软件中)几乎是标配，其对高性能通信(特别是rdma)优化较好。

在DNN 上用处不大，至少目前来看还没有用起来，原因在于大部分的neural network 的参数没那么多，所以矩阵也就没那么大，而目前结果表明batch size提高对training速度提高不大，batch size 上不去那么每次迭代的计算量也没有那么高。综上最合适的架构是4路titan，就这样也没把flops用满。除非网络结构或者training的形式大变，否则MPI暂时用不上的。（纯cpu解决方案也许需要）
如果是广义的机器学习，那MPI太有用了。只要涉及大矩阵运算的还没有什么能超过MPI的。spark之类的差的太远。

优劣对比

  1. 特定于机器学习任务，mpi 最大的优势有两点。一是 mpi 有一个高性能 allreduce 的实现，底层实现了 tree aggregation. 二是程序可以无缝移植到异构高性能计算环境，例如 infiniband.

  2. MPI的一大优势是支持各种网络硬件，例如infiniband和Intel Omni Path，或者网络拓扑结构，例如cray的dragonfly。通常MPI会做各种针对性的优化从而得到不错的性能。这个优势主要体现在超算上。而对于云，通常使用常见的网络硬件（例如Ethernet）和连接结构（multi-rooted tree），MPI做的优化通常不会有太大效果。

  1. mpi 的速度也是非常快的，比zmq快。我自己手写的消息传输也怎么优化也达不到 mpi 的速度。毕竟Argonne实验室的科学家每天都在研究怎么让 mpich 更快。并且对于很多机器学习任务，例如深度学习，网络传输已经不是瓶颈，这会随着万兆网的普及越来越明显。有人说 mpi 的异步传输比较坑，其实很多人用 mpi 都是直接用 send + recv ，配合一个自己写的 signal queue 就行了。

劣势：

  1. 在云上，MPI的一大问题是容灾。任何一个节点出问题会导致整个任务失败，会导致运营成本增加。

    辩论：其实 mpi 不支持容错问题并不大，因为容错的逻辑本身也应该是开发者自己来搞定，除非你能忍受 hadoop 这样的龟速。不过这里有一个小 trick, 使用 MPICH 需要把 auto-cleanup 关掉。因为 mpi manager 默认是只要一个 task 挂掉，就会杀掉所有进程。

  1. mpi 的缺点也很明显，对于机器学习任务来说，主要是支持的语言接口太少，只能拿 C/C++ 或者 fortran 来写，最近也有一个 java mpi 的实现，不知道性能如何。

  2. 但是如果一个系统自己实现fault recovery，那还需要MPI做什么呢？做通信？——现代后台系统都用基于HTTP的RPC机制通信了，比如和Google的Stubby、Facebook的Thrift、腾讯的Poppy还有Go语言自带的rpc package。做进程管理？——在开源界没有分布式操作系统的那些年里有价值；可是今天（2013年），Google的Borg、AMPLab的Mesos和Yahoo!的YARN都比MPICH2做得更好，考虑更全面，效能更高。

结论和建议

  1. 如果要在云上跑机器学习的话，MPI前景不大。但如果是使用Top500类似的超算，MPI是不错的选择。
  2. MPI是分布式领域里的汇编语言，Hadoop是分布式领域里的C语言，Spark是分布式领域里的Java语言。从接口的易用性上，MPI < Hadoop < Spark；从控制的精细程度上，当然是MPI > Hadoop = Spark；性能方面，一个编写良好的的MPI程序肯定是比Hadoop和Spark快的，但是代码量巨大，非常繁琐。

怎样学习才高效

​ 学习是有方法的，而且不同的方法在学习中产生的学习效率不一样。每个人适用的学习方法不一定相同，首先找到适合自己的学习方法，然后持续使用直到形成习惯，你会发现习惯的力量是如此强大。文章标题是高效，那么就必须讨论讨论科学。什么样的学习方法是科学的，是合乎规律的。

1. 科学的学习方法

​ 一个适合自己的学习方法，需要满足的点非常多，而且要通过不断的实验修改细节。不管现在是高中还是初中，以后学习的道路都非常长，一种适用自己的学习方法万一能形成习惯都能直接或间接地影响一生。下面我直接介绍一种常用的学习套路，最后再讨论其他的学习方法。

2. 打造学科基础-知识的脉络

​ 这一步要做到的目标是了然于心。关键在于思维导图，这是一种非常好的归纳工具，而且建议手绘。在学习的过程中不断补充思维导图，然后不断的记忆，按照张宇的说法就是背。无论是数学也好，化学、物理、历史甚至是一门编程语言，一种新的思想，都是这样。

​ 针对这个图进行训练，正着推，从根节点到主干到分支一直推到最后一个的节点。然后开始逆着推，找到一个节点，然后一步步推导回根节点。当然，这个过程是在脑海中推演，顺便培养细节和推演能力。做到这一步，其实对考试来说效果并非立竿见影。接下来就说一个科学的复习方法。

​ 概率论与数理统计。当然不是说这门课，而是要把这种手段利用到考试中去。先说一种统计规律，高考和考研甚至是公务员考试，题目难易程度中下的占比大概是60%-70%。考试部分知识点在考试中出现的概率大于85%，其中相对简单的题型大概占70%，根据上下文也就是近几年出题情况，我们几乎可以把今年考中概率大于90%的题目定位到，这就叫事半功倍。功夫不负有心人，什么叫有心人，瞎学肯定不是。当然，近几年考试肯定是不够的，概率论上来讲，数据量越大，你得到的统计结果越接近真相，至少得十年以上的考题，特别是数学。像历史或者经济学这种学科在打造知识脉络这个阶段已经有了非常好的效果了。

​ 上面说的方法叫我称之为知识脉络，姑且说这是一种科学的复习方法，效果会是怎样呢 ？假设你已经做到了上面三步，毫无疑问你对整个学科的考纲考试的考法，知识脉络都已经比较熟悉，借用一句话，了然于胸，但是你用不出来，就像是一身内功只会一招直拳。所以你需要不断不断不断的练习。

3. 持续不断的训练-快速的解题

​ 磨合。在这个阶段就是学习各种招数，怎么学？就是把各个门派的招数拿出来演练一下。当然应该是从指定的参考书开始，课本后面的练习题，那都是非常经典的。这里面有一个诀窍是质和量那个更重要的问题。一般参考书会将同样类型的题目归结到一起，先仔细研究好例题，完整地完成少量的题目之后就可以开始刷题。拿起来，十秒钟有思路了，放下，没有思路开始研究套路。一直这样循环直到所有辅导书刷过一遍，过了这个知识点，在你的思维导图上面标注。一句题外话，除了学校发的试卷和作业之外，每门功课买三本辅导书是比较基本的，这保证你的题量充足，这算不上题海战术，特别是数学这些非常考验思路跟计算能力的科目，既要保证准确率又要保证速度，怎么做到的？台上一分钟台下十年功，一千题那都是两个月的量而已，不然怎么考清华？

​ 再说说为什么普通院校的学生考不过顶尖院校的学生。顶尖院校的资源充足，有大把优秀的老师把这些年的考试的套路整理好了，因为本来就是985 的学生，这些套路他当年高考的时候就用过。本来这些套路不是普适的，老师之间交流之后，默默把这些套路变成了学校里面的试题..所以其实一个学校的模拟卷的水平其实代表了这个学校老师的实力，代表了这个学校的地位。但是，光靠学校的试题，题量是不足的，平时很多试卷老师也会偷懒，就是在题库里面抽题出来，所以要学会自己筛选题目。

​ 筛选题目要用一个什么样的标准？真题。真题是非常珍贵的，注意我用到了珍贵这个词，很多考试开始到现在时间跨度不过二、三十年，也就是说只有不到三十张卷子，所以一般是反复地、仔细地、不断地做，过一段时间就做一次，当然要留下近三年的真题。这三年的真题用来考前检验自己。

4. 知识归纳-十秒钟内找到思路

​ 知识在运用的时候不熟练导致的问题的使用的时间会很多，零散的知识点在大脑内检索的速度是非常低的。大量的实验表明，记忆都是通过联系产生的。不知道同学们有没有发现，要一下子记住一千个单词很难，但是背诵一片三五千字的文章倒是比较简单。所以，根本的原因的我第一点提到的架构。思维导图建立之后，你想用笔记本去维护一个图是非常难的，因为有大量的知识点你要往里面补充，但是，大脑不一样。把这些知识点往大脑里面补充的时候是一个回忆的过程，这种回忆其实就是检索的过程，你要找到你这个知识点的上一级的知识点是非常快的。一旦建立了这么一个架构之后，你复习起来速度是非常快的。想要十秒钟内找到思路，做到这一步还不够。

​ 要掌握这个学科分析问题的基本方法。每一个学科最基本的定理，课本一般会最直接给出来，但这些东西没法直接结题，在经过训练和填补知识体系架构之后，要进行总结，摸清楚这种类型的题用的是哪个基本定理，要思考那种方法效率是最高的，有多少种方法可以替补。考场上结题速度最快的那些人，其实并不是智商比普通的同学高出多少，而是在经过一系列的训练，有意识的思考和总结之后，这些人形成了一定的条件反射和思考惯性，极大提高了时间优势，而在考场上，唯快不破。李小龙就是把这个方法发挥到极致的代表，几乎每一拳都是一种肌肉记忆和本能反应。无论什么样的考试基本都离不开这个套路，比的就是谁比谁有心。

​ 学而不思则罔，思而不学则殆。共勉。

浅论自由

​ 突发奇想,想分享一下对自由的看法,特别是思想上的自由. 关键词是愤怒,恐惧,懒惰…

​ 我觉得是懒了,思维上懒惰了.在冥想了二十分钟之后,我还是解不开这个心结,文字的能力渐渐变差,对美的记忆也越来越萎靡了,按照这种轨迹发展,那么我想要的自由呢 ? 人性本来就是懒惰的,我深刻的知道这是动物本能,但是人某种维度上不等于动物,我在想,我们如何才能破碎这种几千年来在我们的血脉里盘桓不去的桎梏呢 ? 所谓本能这种东西只有在特殊的时候才会从身上涌现出来,比如说愤怒,比如说空虚,比如说疲惫,在你不经意之间,仿佛潜意识那般决定了你的行为. 技术革命,社交便利之后,出现了一个名词叫标签….所以控制这种本能在现代社会显得十分重要.

​ 当然,控制并不单是对自己人设的搭建维护,更重要的是自我精神修养.能指挥自己行为的思想才是自由的旗帜,一直以来,精神自由都是自我追求最高的境界.
